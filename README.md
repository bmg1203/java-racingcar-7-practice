# 🏎️ 자동차 경주

# ⛳️ 학습 목표

- 여러 역할을 수행하는 큰 함수를 단일 역할을 수행하는 작은 함수로 분리한다.
- 테스트 도구를 사용하는 방법을 배우고 프로그램이 제대로 작동하는지 테스트한다.
- [1주 차 공통 피드백](https://docs.google.com/document/d/1MdiqBV5nhSfFB96-p5LlKrGM8uLopXslh1vEzwxR9Bo/edit?usp=sharing)을 최대한 반영한다.

# 💡 기능 요구 사항

- 주어진 횟수 동안 `n`대의 자동차는 전진 또는 멈출 수 있다.
- 각 자동차에 이름을 부여할 수 있다. 전진하는 자동차를 출력할 때 자동차 이름을 같이 출력한다.
- 자동차 이름은 쉼표(`,`)를 기준으로 구분하며 이름은 5자 이하만 가능하다.
- 사용자는 몇 번의 이동을 할 것인지를 입력할 수 있어야 한다.
- 전진하는 조건은 0에서 9 사이에서 무작위 값을 구한 후 무작위 값이 4 이상일 경우이다.
- 자동차 경주 게임을 완료한 후 누가 우승했는지를 알려준다. 우승자는 한 명 이상일 수 있다.
- 우승자가 여러 명일 경우 쉼표(`,`)를 이용하여 구분한다.
- 사용자가 잘못된 값을 입력할 경우 `IllegalArgumentException`을 발생시킨 후 애플리케이션은 종료되어야 한다.

# 📋 기능 목록

## 자동차 이름 목록 입력 안내 메시지 출력

- [x]  `"경주할 자동차 이름을 입력하세요.(이름은 쉼표(,) 기준으로 구분)"`를 출력한다.

## 자동차 이름 목록 문자열 입력

- [x]  사용자로부터 문자열을 입력 받는다.
- [x]  입력된 문자열이 `null`인지 확인한다.
- [x]  입력된 문자열의 앞 뒤 공백을 제거한다.
- [x]  입력된 문자열이 빈 문자열(`""`)인지 확인한다.
- [x]  쉼표(`,`)를 기준으로 자동차의 이름을 나눈다.

## 자동차 이름 입력값 확인

- [x]  자동차의 이름이 공백으로만 이루어졌는지 확인한다.
- [x]  자동차의 이름의 앞 뒤 공백을 제거한다.
- [x]  자동차의 이름이 빈 문자열(`""`)인지 확인한다.
- [x]  자동차의 이름이 5자를 넘기는지 확인한다.
- [x]  자동차의 이름이 중복인지 확인한다.

## 시도 횟수 입력 안내 메시지 출력

- [x]  `"시도할 횟수는 몇 회인가요?"`를 출력한다.

## 시도 횟수 문자열 입력

- [x]  사용자로부터 문자열을 입력 받는다.
- [x]  입력된 문자열이 `null`인지 확인한다.
- [x]  입력된 문자열의 앞 뒤 공백을 제거한다.
- [x]  입력된 문자열이 빈 문자열(`""`)인지 확인한다.
- [x]  입력 받은 문자열이 `long`의 범위인지 확인한다.

## 입력 받은 시도 횟수 동안 전진 및 정지 실행

- [x]  0에서 9사이에서 무작위 값을 생성 한다.
    - [x]  값이 4 이상일 경우 전진
    - [x]  값이 4 미안일 경우 정지
- [x]  누적 전진 횟수를 저장한다.

## 한번의 시도가 끝나면 실행 결과 출력

- [x]  `“실행 결과”`를 출력한다.
- [x]  실행 결과를 `"(이름) : (전진 횟수)"` 방식으로 출력한다.
    - [x]  `(전진 횟수)`에 전진 1회당 `-`을 출력한다.

## 모든 시도가 끝나면 최종 우승자 출력

- [x]  최종 우승자는 `“최종 우승자 : (우승자 이름)”` 방식으로 출력한다.
    - [x]  공동 우승자가 있을 경우 우승자 이름을 쉼표(`,`)로 구분하여 출력한다.

# ⚡ 예외 처리 사항

- [x]  입력된 문자열이 `null`이거나 빈 문자열(`""`)인 경우 `IllegalArgumentException`을 발생시키고 "`입력이 없습니다.`" 라는 오류 메시지를 출력한 후 애플리케이션을 종료한다.
- [x]  자동차의 이름이 공백으로만 이루어진 경우 `IllegalArgumentException`을 발생시키고 "`자동차의 이름은 공백일 수 없습니다.`" 라는 오류 메시지를 출력한 후 애플리케이션을 종료한다.
- [x]  자동차의 이름이 빈 문자열(`””`)인 경우 `IllegalArgumentException`을 발생시키고 "`자동차의 이름은 빈 문자열일 수 없습니다.`" 라는 오류 메시지를 출력한 후 애플리케이션을 종료한다.
- [x]  자동차의 이름이 중복되는 경우 `IllegalArgumentException`을 발생시키고 "`자동차의 이름은 중복될 수 없습니다.`" 라는 오류 메시지를 출력한 후 애플리케이션을 종료한다.
- [x]  자동차의 이름이 5자를 넘기는 경우 `IllegalArgumentException`을 발생시키고 "`자동차의 이름은 5자 이하 이하만 가능합니다.`" 라는 오류 메시지를 출력한 후 애플리케이션을 종료한다.
- [x]  입력 받은 이동 횟수가 `long`의 범위가 아닌경우 `IllegalArgumentException`을 발생시키고 "`이동 횟수가 너무 많습니다.`" 라는 오류 메시지를 출력한 후 애플리케이션을 종료한다.
- [x]  입력 받은 이동 횟수가 음수인 경우 `IllegalArgumentException`을 발생시키고 "`이동 횟수는 음수일 수 없습니다.`" 라는 오류 메시지를 출력한 후 애플리케이션을 종료한다.
- [x]  입력 받은 이동 횟수가 소수인 경우 `IllegalArgumentException`을 발생시키고 "`이동 횟수는 소수일 수 없습니다.`" 라는 오류 메시지를 출력한 후 애플리케이션을 종료한다.
- [x]  입력 받은 이동 횟수가 숫자가 아닌 경우 `IllegalArgumentException`을 발생시키고 "`이동 횟수는 숫자만 가능합니다.`" 라는 오류 메시지를 출력한 후 애플리케이션을 종료한다.

# 📦 역할 별 패키지 분리

## [ model ]

### Car

- 자동차의 이름과 전진 횟수를 저장할 객체
- 전진 명령이 떨어지면 전진 횟수 증가

### CarFactory

- 입력받은 문자열을 통해서 자동차 객체를 생성

### RandomNumberGenerator

- 0에서 9사이에서 무작위 값을 생성

### Judge

- 해당 시도에 자동차가 전진 또는 정지 판단
- 우승자 판단

## [ view ]

### InputView

- 사용자로부터 문자열을 입력받음

### OutputView

- 결과나 메시지를 사용자에게 출력

### InfoMessage

- 정보성 메시지를 관리

### ErrorMessage

- 에러 메시지를 관리

## [ controller ]

### RacingController

- 경주의 전체 과정을 관리

## [ validator ]

### RawCarNamesValidator

- 입력받은 자동차 이름들의 원본 문자열이 유효한지 판단

### CarNameValidator

- 자동차의 이름이 유효한지 판단

### MovingCountValidator

- 입력 받은 이동 횟수가 유효한지 판단

## [exception]

### RacingException

## [dto]

### RawCarNamesInput

- 입력 받은 자동차 이름들의 원본 문자열을 저장

### RawMovingCountInput

- 입력 받은 이동 횟수를 저장

- `IllegalArgumentException`을 상속받고, `ErrorMessage`를 인자로 받아 처리
