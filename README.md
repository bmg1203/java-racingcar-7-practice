## 2️⃣ _ 나만의 Git Convention 
```
    <Header>
    [Keyword] : 요약
    - 헤더에는 끝에 .을 찍지 않는다
    - 명령어의 어투를 사용
    - Keyword Type
        - Docs : 문서 관련 내용
        - Feat : 새로운 내용 추가(구현)
        - Fix : 기존 구현 내용 수정
        - Del : 기존 구현 내용 삭제
        - Test : 테스트 코드 관련 내용

    <Body>
    - Header와 한 줄 띄워서 작성
    - 모든 내용은 영어로 작성하려고 노력
    - 자세한 수정 내용에 대해서 작성

    <예시>
    Feat : Add 1 method in ServiceImpl.java

    Implement int getRandomMovement(int num) method
        - Input : integer
        - Output : Integer
        - Main Purpose : get Random Number to decide to Move or not

```
